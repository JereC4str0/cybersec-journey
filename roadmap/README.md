# Roadmap mastery-based — Cybersecurity Red Team Portfolio

> Objetivo: conseguir trabajo en ciberseguridad con evidencia pública verificable, sin depender de certificaciones pagas.
>
> English: Mastery-based roadmap. Progress is gated by topics, labs, tools, writeups, and public deliverables — not by calendar weeks.

## Cómo se usa este roadmap

1. Elegí el **bloque actual** en `blocks.md`.
2. No avanzás por fecha; avanzás cuando cerrás la **exit gate**.
3. Cada bloque debe producir evidencia pública en este repo:
   - notas,
   - cheatsheets,
   - writeups,
   - scripts/herramientas,
   - reportes,
   - proyectos.
4. Todo contenido sensible se sanitiza: sin credenciales reales, sin IPs privadas/externas identificables, sin datos de terceros.
5. Práctica ofensiva solo en entornos autorizados: labs, CTFs, VMs propias, plataformas de entrenamiento.

## Regla de oro

Si un tema no produce evidencia pública o skill demostrable, se estudia como repaso opcional, pero no bloquea el roadmap.

## Referencia externa

- https://roadmap.sh/cyber-security — se usa como checklist técnica externa para detectar huecos, no como programa completo.
- Certificaciones de roadmap.sh: opcionales. No son gate mientras el objetivo sea empleabilidad con portfolio.

## Estructura

- `roadmap/README.md` — este archivo.
- `roadmap/blocks.md` — bloques de dominio, entregables y exit gates.
- `roadmap/progress.md` — estado actual y próximos hitos.
- `plan/` — plan legacy por semanas, mantenido como referencia histórica.

## Idiomas

- Notas internas: español.
- READMEs, writeups principales y reportes finales: español + resumen en inglés cuando sea posible.
- Código y commits: inglés técnico claro.

## Loop de mentoría

Cada bloque se trabaja así:

1. Objetivo del bloque.
2. Teoría mínima necesaria.
3. Práctica tangible.
4. Desafío extra.
5. Entregable público.
6. Revisión: qué salió, qué falló, qué falta.
7. Siguiente bloque solo si la exit gate está completa.
